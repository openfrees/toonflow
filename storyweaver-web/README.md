# 🎬 知剧AI 前端应用 (storyweaver-web)

基于 Nuxt 3 构建的高性能前台系统，采用 SSR/SWR（服务端混合渲染）策略，兼顾极速的 SEO 与客户端交互体验。

---

## 🛠 一、Scripts 运行命令详解

项目 `package.json` 中配置了应对各种环境的构建指令，具体含义如下：

| 命令分类 | 执行脚本 | 作用解释 |
| :--- | :--- | :--- |
| **本地开发** | `npm run dev` | 启动本地开发热更新服务器 (默认加载 `.env.localhost`)。<br>本地开发固定按 `localhost` 模式运行，修改代码会实时生效。 |
| **标准构建** | `npm run build` | 【混合渲染/SSR】默认构建。<br>生成 `.output` 目录，供 Node.js 服启动，使用 `nuxt.config.ts` 里的默认配置，不上生产环境用。 |
| **生产构建** | `npm run build:prod` | 🌟 **【核心】** 生产环境专属构建 (注入 `.env.production`)。<br>打出的 `.output` 包内的接口已锁定为正式网址，部署到服务器就用这个！ |
| **标准生成** | `npm run generate` | 【纯静态/SSG】全量焊死生成。<br>把所有页面变成纯 `.html` 死静态页供 Nginx 直接代理。不推荐动态数据网站使用。 |
| **生产生成** | `npm run generate:prod`| 线上环境全量纯静态打包。<br>与 `build:prod` 区别：它生成的是完整的 `.output/public`，不需要 Node.js 环境即可运行。 |
| **本地预览** | `npm run preview` | 本地测试大招！在执行完 `build` 之后跑这个命令，可以在本地启动一个模拟线上环境的 Node Node 服务，用来测试打包是否有 Bug。 |
| **生命周期** | `npm run postinstall`| 依赖安装完后 Nuxt 自动生成的类型声明等准备工作（不需要手动跑）。 |

## 🚀 二、宝塔面板 SSR 生产环境部署指南 (业内最佳实践)

本指南针对 `npm run build:prod`（SSR/SWR 混合渲染模式）在宝塔面板上的部署流程。

### 步骤 1：本地执行生产打包
在本地代码目录执行：
```bash
npm run build:prod
```
打包成功后，在项目根目录会出现一个隐藏文件夹 `.output`。这个文件夹包含：
- `server` 目录 (SSR引擎所在)
- `public` 目录 (静态资源)
- `nitro.json`

### 步骤 2：宝塔服务器环境准备
登录宝塔面板，前往 **软件商店** 安装以下必备组件：
1. 【必备】搜索安装 **Node.js版本管理器** 或 **PM2管理器**。
2. 在该管理器中，将 Node.js 版本切换至 **v18.0** 或更高版本。
3. 【必备】安装并启动 **Nginx**。

### 步骤 3：上传 `.output` 打包产物
1. 前往宝塔的 **文件** 菜单。
2. 在 `/www/wwwroot/` 目录下新建一个文件夹，如 `/www/wwwroot/storyweaver-web`。
3. 将本地打包出来的整个 `.output` 文件夹，上传到此目录中。
   *最终结构看起来是：`/www/wwwroot/storyweaver-web/.output/server/index.mjs`*

### 步骤 4：通过 PM2 启动 Node 服务
1. 打开宝塔左侧的 **网站** 菜单，找到顶部的 **Node项目** 栏目。
2. 点击 **添加Node项目**：
   - 运行环境：选您装好的 Node 版本
   - 项目目录：选刚才建好的 `/www/wwwroot/storyweaver-web/.output/server`
   - 启动文件：选择里面的 `index.mjs`
   - 项目名称：起个名，如 `aistory-web`
   - 项目端口：默认 Nuxt 是 `3000`（也可以自己配死）
3. 点击提交，看到项目状态为 **运行中**，说明服务已经跑在服务器本地的 `127.0.0.1:3000` 了。

### 步骤 5：配置宝塔 Nginx 反向代理（对外访问）
现在前台在本地跑，我们需要让用户通过域名访问。

1. 到宝塔左侧的 **网站** -> **PHP项目**（或HTML项目）中，**添加站点**。
2. 域名填您的前台线上域名（例如 `www.aistory.com`），根目录随便选（因为一会儿我们要反代掉它），提交建站。
3. 在站点列表找到刚才建的站点，点击右侧的 **设置**。
4. 找到 **反向代理** 选项卡，点击 **添加反向代理**：
   - 代理名称：随意（如 `nuxt-ssr`）
   - 目标URL：填写 `http://127.0.0.1:3000`（这是刚才 Node 跑的本地端口）
   - 发送域名：`$host`
5. 点击 **提交**。

🎉 恭喜！开启反代后，用户只要访问您的前台域名，Nginx 就会自动把请求给到 PM2 里的 Nuxt 引擎，享受秒开的混合静态渲染了！
